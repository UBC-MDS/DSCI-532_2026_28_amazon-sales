# Contributing to the Amazon Sales Dashboard Project

This outlines how to propose a change to the Amazon Sales Dashboard Project. 

### Fixing typos

Small typos or grammatical errors in documentation may be edited directly using the GitHub web interface.

### Prerequisites

Before you make a substantial pull request, you should always file an issue and
make sure someone from the team agrees that it's a problem. If you've found a
bug, create an associated issue and illustrate the bug with a minimal 
[reprex](https://www.tidyverse.org/help/#reprex).

### Pull request process

*  A Git branch must be must be created for each pull request(PR).  
*  New code should follow the tidyverse [style guide](http://style.tidyverse.org) or PEP8 [style guide](https://www.python.org/dev/peps/pep-0008/).
*  Each pull request is required approval from at least 1 other collaborating member prior to merging with the main branch.
*  Pull requests should be kept reasonably small and focused, accompanied by a clear description.

### Code of Conduct

Please note that this project is released with a [Contributor Code of
Conduct](CODE_OF_CONDUCT.md). By participating in this project you agree to
abide by its terms.
